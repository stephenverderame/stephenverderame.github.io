Puzzle Solver

Summer 2018

To learn about machine learning, I decided it would be a cool exercise to try and implement a maze and word-search solver in C++. I based my implementation on 3Blue1Brown’s Recent Videos and learned necessary topics in computer vision and linear algebra along the way.

The code for this project is a mess, and I just keep this project on my site as an interesting archive artifact.

The mazes were rather straightforward: select a starting and end point and run A* over a greyscale image. The word searches, however, were rather tricky and utilized a feed-forward neural network for optical character recognition.

The overall process was as follows:

  1. Convert image to greyscale
  2. Perform Canny Edge Detection (Guassian Blur + Sobel Edge Detection + Post Processing) to highlight edges
  3. Perfom a Hough Transform on the result of the Canny Edge Detection to identify “grid lines”
  4. Compute the intersection points of the lines detected by the Hough Transform
  5. For each line intersection (which will occur mostly at locations of letters), try to find a bounding box containing some character. Next, to filter out intersections not part of the grid:
  6. Use a red-black tree to sort x values, y values, distance between x values and distance between y values of each bounding box computed from each line intersection
  7. Use sorted inorder-traversal of the trees to compute medians of the above values
  8. Use the sorted lists to find groups of relatively equal spacing between elements. The largest group of equal spacing is assumed to be the search grid
  9. For each bounding box, draw a point at its center and perform the hough transform again to find grid lines and place the bounding boxes into the search grid
  10. Converts the pixels of each bounding box from Matrix to Vector format, and passes the vector through the Neural Network to calculate its predicted letter value
  11. Finds the words specified in the WordSearchDialog by naive brute-force search over the grid

One thing to note is that I chose a tree sort using a red-black tree simply because I wanted to learn how to implement balanced binary trees. This proved quite useful as later in my functional programming class I would learn Osaki’s Red-Black Tree Implementation and be blown away by the reduction in complexity compared to my imperative implementation in this project.
